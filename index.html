Roboter, die wie eine Flüssigkeit fließen.

Compass16-LBM überträgt das Lattice-Boltzmann-Prinzip auf mobile Schwärme. Statt Trigonometrie und FPU: 16 Kompassrichtungen, ganzzahlige Arithmetik, Skalierung S=256. Ergebnis sind kollisionsfreie, emergent fließende Bewegungen — laufzeitfähig auf 8‑Bit‑Mikrocontrollern.

// 01

Ein Schwarm, beschrieben wie ein Gas.

Die klassische Lattice-Boltzmann-Methode (LBM) simuliert Flüssigkeiten durch zwei lokale Operationen: Relaxation (jeder Knoten nähert sich einem lokalen Gleichgewicht an) und Streaming (Verteilungswerte wandern zu Nachbarn). Aus diesen rein lokalen Regeln entstehen makroskopisch die Navier-Stokes-Gleichungen — obwohl kein Knoten sie kennt oder löst.

Compass16-LBM überträgt dieses Prinzip auf mobile Agenten. Jeder Roboter speichert einen 17-Werte-Verteilungsvektor (16 Kompassrichtungen + Ruhe). Statt komplexer Pfadplanung folgen die Roboter ihrem lokalen Gleichgewicht — und genau daraus entstehen fließende, kollisionsfreie Bahnen, automatischer Abstandsausgleich, Spurbildung in Engstellen, und Deadlock-Freiheit.

Was es ist

Eine ganzzahlige LBM-Variante für mobile Schwärme. Keine Trigonometrie, keine Floating-Point-Operationen, keine zentrale Steuerung. Skalierung S = 256 = 2⁸ erlaubt Shift-Operationen statt Multiplikationen.

Zielhardware: 8‑Bit‑MCUs (ATmega328 & Co.), 16‑Bit‑Systeme. Speicherbedarf pro Agent: 38 Bytes. Rechenkosten pro Zeitschritt: ~276 Zyklen.

Was es nicht ist

Keine Fluidsimulation. Die LBM-Mathematik wird übernommen, aber Begriffe wie „Dichte", „Druck" oder „Schallgeschwindigkeit" werden im Schwarmkontext anders gefüllt — siehe Begriffstabelle weiter unten.

Kein zentraler Planer. Jeder Agent kennt nur seine ~3 nächsten Nachbarn. Globales Verhalten — Spurbildung, Engstellenfluss, Schwarmkohäsion — entsteht emergent aus der Relaxation.

// 02 · Kernkapitel

Die Fluiddynamik der Roboter.

Jeder Roboter speichert einen Vektor f = (f₀, f₁, …, f₁₆) — eine ganzzahlige Bewegungstendenz in jede der 16 Kompassrichtungen plus Ruhe. Dieser Vektor wird in jedem Zeitschritt durch sechs Phasen geschoben. Aus den daraus entstehenden lokalen Regeln entstehen die makroskopischen, fließenden Eigenschaften des gesamten Schwarms.

Compass16-Gleichgewicht (Herzstück)
feqk  =  wk · ( ρ  +  κ · ( ck · utarget ) )
Der Sollzustand jedes Agenten. Die Relaxation zieht den aktuellen Vektor fk kontinuierlich in Richtung dieses Gleichgewichts. Die Sollbewegung utarget setzt sich zusammen aus Navigation, Separation und Kohäsion (Reynolds-Boids in Integer-Form).

// Die fünf emergenten Eigenschaften

01

Fließverhalten um Hindernisse

ω ≈ 128 · β > α

Bei moderater Relaxation und starker Separation weichen Agenten Hindernissen aus wie eine Flüssigkeit, die um einen Stein fließt. Die Verteilungswerte verteilen sich automatisch auf die freien Compass-Sektoren — kein expliziter Ausweichpfad muss berechnet werden.

→ emergent, nicht programmiert
02

Automatischer Abstandsausgleich

d² < d²ₘᵢₙ → ⊕ · d² > d²ₘₐₓ → ⊖

Hohe lokale Agentendichte erzeugt starke Separationsbeiträge, die Agenten auseinandertreiben. Niedrige Dichte erzeugt Kohäsionsbeiträge, die zusammenziehen. Der Gleichgewichtsabstand stellt sich selbstorganisiert ein — wie Moleküle in einer Flüssigkeit.

→ Gummiseil-Verhalten mit Ruhelänge
03

Geordneter Gegenverkehr

Bounce-Back · fk ↔ fk̄

In schmalen Passagen erzeugt der Bounce-Back an Wänden eine asymmetrische Verteilung. Aus dieser Asymmetrie folgt natürlicher Gegenverkehr mit impliziter Spurbildung — ohne dass eine Verkehrsregel programmiert wäre.

→ antiparallele Reflexion der Verteilung
04

Deadlock-Freiheit

uZiel ≠ 0 ⇒ ρ-Fluss permanent

Die Ruhe-Verteilung f₀ wird durch Relaxation ständig zum Gleichgewicht gezogen. Da das Gleichgewicht eine Zielrichtung enthält, solange das Ziel nicht erreicht ist, kann kein Agent dauerhaft stehenbleiben. Das System bewegt sich immer.

→ keine Patt-Situationen
05

Viskosität ⇄ Trägheit

ν ∝ (S/ω − ½)

Der Relaxationsparameter ω steuert die Trägheit der Richtungsänderung — exakt analog zur Viskosität in der LBM. Klein (ω≈32): honig­artig, vorausschaubar. Groß (ω=256): wasserartig, abrupt. Standardwert ω=128 ergibt glatte, fließende Bewegung.

→ flüssigkeits­ähnliches Spektrum per Slider
06

Pheromon-Kanal (optional)

ACO-Schicht · 🐜

Zusätzlich zum LBM-Kern hinterlässt jeder Agent in einem Gitter eine richtungsbehaftete Pheromonspur. Andere Agenten lesen diese und mischen sie in utarget. Resultat: Trampelpfade entstehen über die Zeit ohne explizite Pfadplanung.

→ ACO-inspirierte emergente Wegfindung

// 03

Warum 16 Richtungen, nicht 8?

Die klassische LBM verwendet D2Q9 — 8 Richtungen plus Ruhe. Für mobile Agenten ist das zu grob: schon 22,5°-Sprünge wirken eckig. Compass16 verdoppelt die Auflösung auf 16 Richtungen mit Δ = 22,5°, bleibt aber vollständig integer-kompatibel.

// COS / SIN

Vorberechnete Konstanten

Alle 32 Komponenten ck,x, ck,y sind ganzzahlig (max ±256), einmalig im ROM gespeichert. Beispiel: NNE = (98, −237).

// SHIFT >> 8

Skalierung S = 256 = 2⁸

Jede Division durch S wird zu einem 8‑Bit‑Right-Shift. Multiplikationen mit Steuerparametern (κ, α, β, γ) werden durch Wahl als Zweierpotenz zu Left-Shifts.

// k̄ = ((k−1+8) % 16) + 1

Antiparallelität in O(1)

Für jede Richtung k existiert exakt der antiparallele Index k̄. Bounce-Back wird damit ein einziger Vertausch — ohne Trigonometrie und ohne Vergleichskaskade.

// 17 × 2 BYTES

38 Bytes pro Agent

17 × 16-Bit-Verteilungswerte (34 B) plus 2 × 16-Bit-Position (4 B). Auf einem ATmega328 mit 2 KB SRAM: ~50 Agenten gleichzeitig möglich.

// ~276 ZYKLEN

~58 Hz @ 1000 Agenten

Auf 16 MHz: 1000 Agenten in 17,3 ms pro Schritt. Reynolds-Flocking mit √ und arctan benötigt 8–10× mehr Zyklen für dieselbe Aufgabe.

// τ₁=51 · τ₂=171

Klassifikation per Schwellwert

Die Compass16-Klassifikation arbeitet mit zwei festen Schwellwerten (51/256 und 171/256). Eingaben (Δx, Δy) werden in 12 Zyklen in einen Sektor 1…16 abgebildet — ohne atan2.
